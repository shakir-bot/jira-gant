# Гант + Jira — инструкция для пользователей

Диаграмма Ганта, которая работает прямо у вас на компьютере и напрямую
общается с вашей Jira. Устанавливать Node.js или что-либо ещё не нужно —
всё уже внутри одного файла.

---

## Шаг 1. Скачать приложение

1. Откройте страницу последней сборки:
   **https://github.com/shakir-bot/jira-gant/actions**
2. В списке выберите самый верхний (свежий) успешный запуск
   («Build portable executables», зелёная галочка)
3. Внизу страницы, в разделе **Artifacts**, скачайте нужный архив:
   - **gantt-jira-win** — если у вас Windows
   - **gantt-jira-mac** — если у вас Mac
4. Распакуйте скачанный ZIP-архив в любую папку (например, «Загрузки»
   или Рабочий стол)

> Понадобится обычный аккаунт GitHub, чтобы скачать файл со страницы Actions
> (если своего нет — попросите ссылку на файл у того, кто уже скачал).

---

## Шаг 2. Запустить

**Windows:**
- Найдите файл `gantt-jira-win-x64.exe` внутри распакованной папки
- Дважды кликните по нему
- Если Windows покажет синий экран **«Windows защитила ваш компьютер»**
  — это стандартное предупреждение для непопулярных программ, не
  вредоносное ПО. Нажмите **«Дополнительно»** → **«Выполнить в любом
  случае»**
- Откроется чёрное окно консоли — не закрывайте его, пока пользуетесь
  приложением

**Mac:**
- Откройте Терминал (Программы → Утилиты → Терминал)
- Перейдите в папку с файлом, например:
  ```bash
  cd ~/Downloads
  ```
- Снимите блокировку macOS и дайте файлу права на запуск (нужно один раз):
  ```bash
  xattr -d com.apple.quarantine gantt-jira-mac-arm64
  chmod +x gantt-jira-mac-arm64
  ```
  (если у вас Intel-Mac, а не Apple Silicon — используйте файл
  `gantt-jira-mac-x64` вместо `arm64`)
- Запустите:
  ```bash
  ./gantt-jira-mac-arm64
  ```

После запуска в консоли появится:
```
Гант + Jira запущен:
  на этом компьютере:  http://localhost:3210
```

---

## Шаг 3. Открыть в браузере

Перейдите на **http://localhost:3210** в любом браузере.

---

## Шаг 4. Создать API-токен Jira (один раз)

Вход в приложение — не через пароль, а через персональный API-токен. Это
официальный способ Atlassian для сторонних инструментов: токен даёт доступ
только к API, не «расшаривает» пароль, и его можно в любой момент отозвать.

1. Откройте **https://id.atlassian.com/manage-profile/security/api-tokens**
   (нужно быть залогиненным в свой аккаунт Jira/Atlassian)
2. Нажмите **Create API token**
3. Дайте токену любое понятное название, например `gantt-app`
4. Нажмите **Create**
5. **Сразу скопируйте показанный токен** — второй раз посмотреть его будет
   нельзя (только создать новый)

> Токен — как пароль от конкретного сервиса. Никому не пересылайте его в
> открытом виде (мессенджеры, почта) и не публикуйте. Если токен всё же
> куда-то «утёк» — зайдите на ту же страницу и нажмите **Revoke**, чтобы
> его отозвать, потом создайте новый.

## Шаг 5. Войти в приложение

На странице `http://localhost:3210` заполните:

| Поле | Что указать |
|---|---|
| **Адрес Jira** | `detmir.atlassian.net` (можно без `https://`) |
| **Email** | тот, под которым вы обычно входите в Jira |
| **API-токен** | токен, скопированный на шаге 4 |

Поставьте галочку **«Запомнить на этом устройстве»** — тогда при следующем
запуске приложения вводить данные заново не придётся.

Нажмите **Войти**.

---

## Шаг 6. Выбрать фильтр и работать

После входа — экран выбора фильтра Jira (избранные показываются сразу,
остальные ищутся по названию). Выберите нужный — построится диаграмма.

Что доступно на диаграмме:
- Перетаскивание задач мышкой — меняет плановые даты
- Растягивание за края бара — меняет длительность
- Правый клик по задаче — связи между задачами, назначение в этап, удаление,
  открыть в Jira
- Кнопка **➕** — добавить новую задачу вручную или новый этап (список
  всех этапов с переименованием/удалением — чипы «Этапы:» над диаграммой)
- Кнопка **📍 + Веха** — добавить веху (цветной флажок) на график
- Вкладка **👥 Команда** — сотрудники, их загрузка и отпуска (используется
  для автоматического прогноза сроков задач без оценки в Jira)
- Вкладка **📅 Календарь** — обзор по неделям/месяцам/кварталам
- Клик по статусу/исполнителю прямо в таблице — редактирование с
  синхронизацией обратно в Jira

## Частые проблемы

**«fetch failed» / «Не удалось подключиться к Jira по сети»**
Скорее всего корпоративная сеть требует прокси или подменяет сертификаты
сайтов (частая практика антивирусов/фаерволов компаний). Решение — задать
переменную окружения `HTTPS_PROXY` (для прокси) или `NODE_EXTRA_CA_CERTS`
(для сертификата). Подробная инструкция с картинками — в README проекта:
https://github.com/shakir-bot/jira-gant#корпоративный-прокси

**«Jira отклонила email/токен»**
Проверьте, что скопировали токен полностью (без пробелов в начале/конце) и
что email указан именно тот, под которым вы входите в Jira (не личная почта,
если для входа используется рабочая).

**Как остановить приложение**
Закройте чёрное окно консоли (Windows) или нажмите `Ctrl+C` в Терминале (Mac).

**Как обновиться на новую версию**
Скачайте свежую сборку с той же страницы Actions и замените старый exe-файл.
Ваши данные для входа и локальные правки (даты, связи, вехи, этапы)
сохранены отдельно и не пострадают.
